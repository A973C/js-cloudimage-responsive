Demo Cloudimage.io Responsive!

Cloudimage responsive plugin allows you to load only the exact image size you need supporting different pixel ratio devices


Quickstart

Include cloudimage.io-responsive.min.js file into your project (after all your content in body, before other scripts):
<script src="https://scaleflex-global.airstore.io/cloudimg/responsive/plugin.min.js"/></script>

Initialize jScaler with your token information (put it right after cloudimage.io-responsive script)
document.addEventListener('DOMContentLoaded', function() {
  jScaler.process({
    TOKEN: '[your token]', // required
    BASE_URL: '[your image folder on server]' // optional | e.g. "https://gallery.airstore.io/"
  });
});

Find your img tags and replace src="[image source]" to ci-src="[image source]"
<img ci-src="birds.jpg">

Also works for background (just add the link to image as ci-img-background attribute):

<div class="background-styles" ci-img-background="birds.jpg"></div>

Configuration params

      TOKEN:          '******',  - Token
      DEFAULT_WIDTH:  '400',     - Default width of images
      DEFAULT_HEIGHT: '300',     - Default height of images
      DEFAULT_TYPE:   'width',   - Default resize operation
      DEFAULT_PARAMS: 'none',    - Default parameters
      BASE_URL:       '/',       - Default url for img folder
      PRESETS: {
        xs: 576,                 - 0 - 576      extra small screen
        sm: 768,                 - 577 - 768    small screen
        md: 992,                 - 769 - 992    medium screen
        lg: 1200,                - 993 - 1200   large screen
        xl: 1920                 - 1200 - 1920  extra large screen
      },

      ORDER:   ['xl', 'lg', 'md', 'sm', 'xs'], - Default order
      AUTO:    [1920, 1200, 992, 768, 576]     - Default images width for responsive background

jScaler attributes

ci-src or ci-img-background

required
jScaler identifier takes as value image path (absolute url or relative to BASE_URL).

NOTE: The images should be stored on a server.

ci-size

optional, default: 400px
Size of an image which is used as a base for creating retina ready and responsive image element.

Possible values:
  • [width]:

    e.g. ci-size="500" => 500px,750px, 1000px, 1500px width for 1x, 1.5x, 2x, 3x pixel ratio accordingly

  • [width x height]:

    e.g. ci-size="300x400" => 300px/400px, 450px/600px, 600px/800px, 900px/1200px width/height for 1x, 1.5x, 2x, 3x pixel ratio accordingly

  • [Width and height for different screen resolutions]
    e.g. ci-size="{xs: '50x100', sm:'100x125', md: '150x150', lg:'200x175', xl:'300x200'}" =>
    for max-width of screen 576px - 50x100, 75x150, 100x200, 150x300 width/height for 1x, 1.5x, 2x, 3x pixel ratio accordingly
    for max-width of screen 768px - 100x125, 150x187.5, 200x250, 300x375 width/height for 1x, 1.5x, 2x, 3x pixel ratio accordingly
    for max-width of screen 992px - 150x150, 225x255, 300x300, 450x450 width/height for 1x, 1.5x, 2x, 3x pixel ratio accordingly
    for max-width of screen 1200px - 200x175, 300x262.5, 400x350, 600x425 width/height for 1x, 1.5x, 2x, 3x pixel ratio accordingly
    for min-width of screen 1201px - 300x200, 450x300, 600x400, 900x600 width/height for 1x, 1.5x, 2x, 3x pixel ratio accordingly
    You can drop some breakpoints, for example ci-size="{sm:'100x125', xl:'300x200'}":
    for max-width of screen 768px - 50x100, 75x150, 100x200, 150x300 width/height for 1x, 1.5x, 2x, 3x pixel ratio accordingly
    for min-width of screen 1201px - 300x200, 450x300, 600x400, 900x600 width/height for 1x, 1.5x, 2x, 3x pixel ratio accordingly

ci-type

optional, default: width

Resize operations


Possible values:

  • width - to resize with a specific width
  • height - to resize with a specific height
  • crop - to crop the image at the center
  • fit - to resize the image in a box and keeping the proportions of the source image
  • cover - to resize the image in a box without keeping the proportions of the source image

Here, you can find some description and how they work

ci-params

optional, default: none

Parameters are used to apply transformations to the image, or to add options to the result image.


Possible values:

  • fgrey - apply a greyscale filter on the image
  • fgaussian[0..10] - apply a gaussian blur filter on the image
  • fcontrast[-100..100] - apply a contrast filter on the image
  • fbright[0..255] - apply a brightness filter on the image
  • fpixelate[0..100] - apply a pixelate filter on the image
  • fradius[0..500] - create a radius on the corners
