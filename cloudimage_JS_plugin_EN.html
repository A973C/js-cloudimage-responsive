Cloudimage JS - Demo and Plugin

Implementing Cloudimage on your website to resize, compress and accelerate images requires you to change the Image URLs in the img tags in your code.

For example: https://sample.li/birds.jpg becomes https://demo.cloudimg.io/width/350/s/sample.li/birds.jpg

The Cloudimage JS Plugin does it automatically for you. It's light (only 10KB), it's simple, it's fast, it's Cloudimage.

Quick start

Step 1: include cloudimage.io-responsive.min.js file into your project (after all your content in body, before other scripts):
<script src="https://scaleflex-global.airstore.io/cloudimg/responsive/plugin.min.js"/></script>
Step 2: just under the above line, initialize jScaler with your token information and optional parameters
document.addEventListener('DOMContentLoaded', function() {
  jScaler.process({
    TOKEN: '[your token]',                         // required
    BASE_URL: '[your image folder on server]',     // optional | e.g. "https://s3-eu-west-1.amazonaws.com/bucket/"
    DEFAULT_TYPE: 'width',                         // default resize operation
    DEFAULT_PARAMS: 'none',                        // default parameters

    // for advanced use cases
    // you can change default presets and add more breakpoints to suit all your design
    PRESETS: 'xs:576|sm:768|md:992|lg:1200|xl:1920' // default breakpoints
  });
});
Step 3: Replace your img tags fromsrc="[image source]" to ci-src="[image source]" and add the resizing parameters (see below)
<img ci-src="flat1.jpg" ci-size="200">


This also works for background, simply add the link to image as ci-background attribute:

<div class="background-styles" ci-background="flat2.jpg" ci-size="300x100" ci-operation="crop" class="background-styles"></div>
That's it, the JS plugin will modify all your image URLs for them to be resized and delivered by Cloudimage.

jScaler attributes

ci-src or ci-background

required
jScaler identifier takes as value image path (absolute url or relative to BASE_URL).

NOTE: Your images should be stored on a server or storage bucket (S3, Google Cloud, Azure Blob...) reachable over HTTP or HTTPS by Cloudimage. You can also upload your images into Cloudimage, more info here.

ci-size

optional, default: auto
Size of an image which is used as a base for creating retina ready and responsive image element.

Possible values:
PR - stands for your device Pixel Ratio
  • [width]:

    e.g. ci-size="250" => width: 250 * PR (px); height: auto;

    <img ci-src="flat2.jpg" ci-size="250">
  • [width x height]:

    e.g. ci-size="125x200" => width: 125 * PR (px); height: 200 * PR (px);

    • <img ci-src="flat2.jpg" ci-size="125x200" ci-operation="crop">
  • [Width and height for different screen resolutions]
    e.g. ci-size="{xs: '50x100', sm:'100x125', md: '150x150', lg:'200x175', xl:'300x200'}" 
    <img ci-src="flat2.jpg" ci-size="{xs: '50x100', sm:'100x125', md: '150x150', lg:'200x175', xl:'300x200'}" ci-operation="crop">

    for max-width of screen 576px - width: 50 * PR (px); height: 100 * PR;
    for max-width of screen 768px - width: 100 * PR (px); height: 125 * PR;
    for max-width of screen 992px - width: 150 * PR (px); height: 150 * PR;
    for max-width of screen 1200px - width: 200 * PR (px); height: 175 * PR;
    for min-width of screen 1201px - width: 300 * PR (px); height: 200 * PR;
    You can drop some breakpoints, for example ci-size="{sm:'100x125', xl:'300x200'}": 
    <img ci-src="flat2.jpg" ci-size="{sm:'100x125', xl:'300x200'}" ci-operation="crop">

    for max-width of screen 768px - width: 50 * PR (px); height: 100 * PR;
    for min-width of screen 769px - width: 300 * PR (px); height: 200 * PR;

ci-operation

optional, default: width

Possible values:

  • width - to resize with a specific width
  • height - to resize with a specific height
  • crop - to crop the image at the center
  • fit - to resize the image in a box and keeping the proportions of the source image
  • cover - to resize the image in a box without keeping the proportions of the source image

You can find here the full Operations documentation.

ci-filter

optional, default: none

Filters allow you to modify the image's apperance and can be added on top of the resizing features above.

Possible values:

  • fgrey - apply a greyscale filter on the image
  • fgaussian[0..10] - apply a gaussian blur filter on the image
  • fcontrast[-100..100] - apply a contrast filter on the image
  • fbright[0..255] - apply a brightness filter on the image
  • fpixelate[0..100] - apply a pixelate filter on the image
  • fradius[0..500] - create a radius on the corners

You can find here the full Filters documentation.
